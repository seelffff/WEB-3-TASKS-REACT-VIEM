
### **Уровень 1: Основы State & Props (20 задач)**

7.  **Профиль пользователя:** Компонент принимает через props `name`, `bio` и `avatarUrl` и красиво их отображает.
8.  **Аккордеон:** Клик на заголовок разворачивает/сворачивает контент.
9.  **Рейтинг (звёздочки):** Отображай 5 звёзд. При клике на звезду все звёзды до неё (включительно) закрашиваются.
10. **Поиск:** Поле ввода. При вводе текста фильтруй статичный массив строк (например, городов) и отображай совпадения.
11. **Модальное окно:** Кнопка "Открыть" показывает затемнённое модальное окно с кнопкой "Закрыть".
12. **Вкладки (Tabs):** Несколько заголовков вкладок. При клике отображается соответствующий контент.
13. **Форма обратной связи:** Поля "Имя", "Email", "Сообщение". Кнопка "Отправить" выводит данные в консоль.
14. **Кастомный хук useLocalStorage:** Создай хук, который синхронирует state с localStorage.
15. **Загрузка данных:** При монтировании компонента загрузи список пользователей с [JSONPlaceholder](https://jsonplaceholder.typicode.com/) и отобрази его.
16. **Калькулятор:** Простой калькулятор с кнопками +, -, *, / и дисплеем.
17. **Генератор паролей:** Поля: длина пароля, чекбоксы (использовать цифры, спецсимволы). Кнопка "Сгенерировать".

19. **Слайдер изображений:** Компонент показывает одно изображение, кнопки "Вперёд"/"Назад" переключают его.

34. **Кастомный хук useForm:** Создай хук для управления формой, который обрабатывает изменения, валидацию и сабмит.
35. **Редактирование профиля "на месте":** Показывай данные пользователя, при клике на "Редактировать" они превращаются в поля ввода.

39. **Виртуализированный список:** Реализуй отображение очень длинного списка (10000+ элементов) с виртуализацией (используй `react-window`).

82. **Кастомный хук useClickOutside:** Создай хук, который определяет клик вне элемента (полезно для закрытия выпадающих меню).

90. **Управление состоянием формы с useReducer:** Реализуй сложную форму (например, мультишаговую) используя `useReducer` вместо `useState`.
91. **Кастомный хук useLocalStorage с сериализацией:** Улучши хук из задачи 14, чтобы он работал не только с примитивами, но и с объектами/массивами.
92. **Контекст с оптимизацией:** Создай Context для приложения, но так, чтобы компоненты, не использующие его, не ререндерились при его изменении (разбей Context на части или используй библиотеку).

95. **Кастомный хук useId:** Используй встроенный хук `useId` для генерации уникальных id для элементов форм.
96. **Структура папок (Feature-Sliced Design):** Организуй код приложения по feature-based методологии (features, entities, shared).

100. **Мониторинг производительности:** Используй `<Profiler>` или React DevTools, чтобы найти и устранить "бутылочное горлышко" в производительности одного из твоих компонентов.

**Как подходить к решению:**

1.  **Пиши код с нуля.** Не копипасть.
2.  **Используй Functional Components и хуки.**
3.  **Деплой!** После каждой задачи (особенно из уровней 2-4) деплой приложение на Vercel/Netlify. Это твоё портфолио.
4.  **Используй TypeScript.** Это не указано в задачах, но это обязательный навык. Перепиши все задачи на TS.
5.  **Комментируй сложные места.**
6.  **Создай публичный GitHub-репозиторий** для всех решений.
